Git & GitHub

Git is an unavoidable tool that you will need to get used to, so there is no time like the present (not to mention, it is required that you use it for the course). Git is a version control system (VCS) used for tracking changes in source code. If you have never used GitHub, Bitbucket, or GitLab before, then this will be pretty useful to read, although it will not cover everything. If you do not have Git installed on your machine, you can download it here (or use a package manager if you are on Linux/Mac).

GitHub

For the class, you will be using GitHub; if the repositories have not been made yet, they will soon (assuming the teams have been made - it’s hard to plan a few months ahead). GitHub has a feature called Issues which will be very important for you all during the semester; this is partially how we will be tracking the work that you have done.

Basic Git Commands

git clone [url]
This will clone an existing git repository to your machine; to make things easier on you all, the repository will exist ahead of time. This way, you do not have to worry about setting the remote-url.

git add [file]
This will add a file to your next commit; alternatively, if you are using VS Code, all you have to do is go to the Source Control tab on the left and add the files there.
