About JavaScript

JavaScript is weakly (loosely) typed language and is essential for developing web applications. It allows the sites to be interactive and not static like this one. JavaScript is ‘object-based’, although there are 7 primitives (variable types that are not objects). These primitives are string, number, bigint, boolean, null, undefined, and symbol. With the exception of null and undefined, all other primitives have an object equivalent that wrap around the primitive. If the wrapper needs to be applied, JavaScript will actually do this for you. Note that primitives are immutable, so using one of the wrapper methods for a primitive will return a new instance of the primitive, but it will not modify it.

In addition to these 7 primitives, JavaScript also has constants. Constants cannot be reset, but constants can change their instantiation.

JavaScript was originally only for client-side changes and making websites dynamic, but it is used for much more now. With things like Node.js and Electron, full desktop applications can be made using JavaScript.

In JavaScript, you don’t need to put semicolons at the end of lines / statements, but it makes your code a lot easier to read if you do, so I recommend it. Also, issues can come up if you don’t use semicolons because JavaScript has to guess where they go. I recommending reading this blog post for a pretty good explanation.

JavaScript is also an asynchronous programming language. This can be especially hard to wrap your head around if you have never dealt it before. JavaScript now follows the async/await syntax, similar to Python and C#. It also uses something called Promises, but I will go into more depth with that later; gotta walk before you can run. You will more than likely have issues this semester where you will need to know how handle JavaScript being asynchronous.

Hello World

Printing to console in JavaScript is extremely easy, and if you have ever coded in C# or Java before, it will look familiar. For those of you who haven’t, it might look a little weird:

console.log("Hello World");

The console itself is an object. There are several methods available to the console that you can use. The most popular ones you will use are clear(), error(), and log(). console.log("") will print to stdout, and console.error("") will write to stderr (if you are running JS in a terminal).

Printing to the console in web applications will only be useful if you are actually looking at the console. You can do this by using the inspect tool in Chrome and clicking on the ‘console’ tab.

Assigning Variables

JavaScript is dynamically-typed like Python, which means you do not have to specify the type like you are used to. In C++ if you wanted to create a variable with a value of “hello”, you would have to do something similar to this:

std::string a = "hello";

In JavaScript, this is how you would assign it: 

var a = "hello";

The variable a is of type string, but you did not have to specify that. Similarly, if you used the same variable a and assigned it to a value of 5, its type would now be number. This can be confusing, especially when you have a member variable of a class and are expecting it to be one type and it returns another. TypeScript is a typed version of JavaScript, and it solves this issue. Although it compiles down to JavaScript, I prefer using it over JavaScript.

Comparison

Most comparison operators are the same, but there are a few differences in JavaScript from C++.

  • == - equals, must match value
    • 5 == “5” (true)
  • === - equals, must match value AND type
    • 5 === “5” (false)
  • != - not equals, must not match value
    • 5 != “5” (false)
  • !== - not equals, must not match value AND type
    • 5 !== “5” (true)

I recommend using === and !==. If you don’t care about the type, you are probably doing something wrong.

Interpolated Strings

Interpolated stings - a string literal that might contain interpolation expressions

This is something easier to explain with code other than words:

const i = 42;
console.log(`i = ${i}`);
console.log("i = ${i}");

The above will print out:

i = 42
i = ${i}

Instead of using double or single quotes, you must use ‘back-quotes’ (the key next to one (below the tilde ~) `).

Arrays

Arrays in JavaScript are more so like vectors in C++ than they are arrays. For starters, arrays do not have a type to them, meaning you can have numbers, strings, objects, and whatever else you want in the same array. Take a look at this example with arrays:

var a = ["this", "is", 1, "array"];
console.log(a[1]);

// Let's add something to the array
a.push("something");
console.log(a);

// Now let's take it away
var i = a.pop();
console.log(i);
console.log(a);

// Print off the number of elements in the array
console.log(a.length)

Classes

Classes are probably going to be one of the most useful things for you. You can create a class doing something like this:

export class Car {
    constructor(make, model, year, color) {
        this.make = make;
        this.model = model;
        this.year = year;
        this.color = color;
    }
}

You’ll notice that I have export in front of the class. Typically, you should be making a class in its own file. This keeps everything separated and easier to deal with, especially working with a team. The export allows the import statement that you see down below, where I am creating an instance of a Car in another file:

import {Car} from "path_to_car";

var a = new Car("BMW", "M5", 2000, "Carbon Black");
console.log(a);

Car {
    make: "BMW",
    model: "M5",
    year: 2000,
    color: "Carbon Black"
}

You can additionally add several other things to the class, such as getters and setters. Classes are very similar to what you have used them for in C++ in previous courses, although you will get a lot of use of them out using JavaScript, especially when you are dealing with data from a database.

Exceptions and Error Handling

You have probably never been taught this, but C++ has something called a “try-block”. Essentially, a section of code will “try” to run. If any exceptions are caught, it will go to the “catch” section. JS has the same feature:

try {
    functionIDidNotWrite();
}
catch(e) {
    console.error(e);
}
finally {
    console.log("This will run no matter what");
}

The code in the try section will try to run; if the function throws an Exception, it will run the section of code in the catch block of code. The finally section will run whether an exception is thrown or caught.

You should get very comfortable with try-blocks, as almost every programming language has them. Up to this point, most of the “error-handling” you have done has probably been with cout or printf. Using try-blocks will be very beneficial to you as you continue to program.
